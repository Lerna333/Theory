Монолитная архитектура (monolithic architecture, MA) - архитектура, в которой все ПО разрабатывается как единый неделимый блок. Все компоненты взаимозависимы и взаимосвязаны. Для запуска приложения необходим каждый компонент приложения.

Микросервисная архитектура (microservices architecture, MSA) - архитектура, которая состоит из отдельных автономных модулей (микросервисов). Каждый микросервис выполняет одну функцию и отвечает за одну бизнес-задачу. Взаимодействие между модулями происходит через API.


Разница между разница между микросервисной и монолитной архитектурой:

Монолит: 
1. Представляет собой единый неделимый блок.
2. Все приложение разворачивается как единое целое.
3. Приложение масштабируется целиком.
4. Используется одна технология/ЯП на весь проект.
5. Изменения, вносимые в одну часть приложения, могут затронуть все приложение.
6. Сбой в одном модуле может привести к остановке всего приложения.


Микросервис: 
1. Состоит из отдельных модулей.
2. Каждый сервис можно развернуть отдельно.
3. Можно масштабировать отдельный сервис.
4. Разные сервисы могут использовать разные технологии/ЯП.
5. Изменения в одном модуле могут не влияют на другие.
6. Сбой в одном модуле может не влиять на работу других модулей.
